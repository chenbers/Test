2 Authentication and Permissions

To use any request a user must be a registered user in the system and must have "Admin" role.

Use http:// or https:// and the host and port that was supplied to you and add the path and parameters as specified below. An example is: http://my.inthinc.com/service/api/vehicles

See Data Dictionary for the formats for data in request bodies and response entities.

4 Status Codes

4.1 Success Codes

4.1.1 200 OK

The request was successful.

4.1.2 201 Created

The request was successful and a new resource has been created. The response entity should contain either the created resource or a link to the created resource or resources.

4.1.3 304 Not Modified

The request was successful but nothing was changed. Results from an update where no records are actually changed.

4.2 Error Codes

4.2.1 400 Bad Request

The request was not formatted correctly.

4.2.2 401 Unauthorized

Users must log in, or include valid authorization credentials in the request.

4.2.3 403 Forbidden

The user is forbidden from using this request. Users must have Admin role (set up in the portal) to use this API

4.2.4 404 Not Found

The server can't find anything matching the request URI.

4.2.5 500 Internal Server Error

The server encountered an unexpected condition which prevented it from fulfilling the request.

4.2.6 501 Not Implemented

The server does not support the functionality required to fulfill the request.

5.18 Time Zones


We use the Olsen Time Zone Database (tz database) as used by Java.


Time zone representation is described at the W3C Working Group site Working with Time Zones


Examples of valid time zone representations in the timeZone properties are:


<timeZone>US/Mountain</timeZone>


<timeZone>Africa/Johannesburg</timeZone>


The short codes, such as MDT and SAST are not valid in requests (they are deprecated in Java because they are not unique) though they may appear in response entities.


5.19 Locales


Locales are represented by the language code and optional variant code.


We currently support:


<locale>en_US</locale> (American English)
<locale>ro</locale> (Romanian)
<locale>es</locale> (Spanish)
<locale>fr</locale> (French)
