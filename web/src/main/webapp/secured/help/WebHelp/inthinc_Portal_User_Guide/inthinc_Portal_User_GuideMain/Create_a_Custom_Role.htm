Create a Custom Role

Part of the task of creating a user in the system is defining that user’s role or roles in the organization, which affects the rights that user has to view and edit information in the system. The standard roles offered include Admin and Read-Only User. Admin is only assigned to a few individuals in an entire organization who supervise the use of the tiwiPro™ Portal. Every other User added to the system is only granted Read-Only rights for their assigned group by default.

To delegate the authority to maintain portal information beyond the All or Nothing rights of Admin and Read-Only User, you can create custom roles, such as Division Head, Manager, Team Lead, and Clerk, to grant selected Read/Write rights over portion of the portal.

Note: A user in the system is assigned one Group, but can be assigned multiple roles. The role assigned to a user applies throughout the group that they are assigned. So the higher the assigned group in the company, the more authority the user has.

For example:

•   A Division Head role may be created with all access points enabled. When a user in Division B is assigned the role of Division Head, the user can Read/Write/Edit information throughout the system for Division B only.

•   A Manager role may be created with these access points enabled: Users, Vehicles, Devices, Zones, Red Flags, and Reports. When a user in the North Team is assigned the role of Manager, that user can Read/Write/Edit the Users, Vehicles, Devices, Zones, Red Flags, and Reports for North Team only. They will have Read-Only rights to the non-enabled access points of North Team.

•   A Team Lead role may be created with only Vehicles and Devices access points enabled. When a user on the North Team is assigned the role of Team Lead, that user can Read/Write/Edit just Vehicles and Devices in their assigned group. They will have Read-Only rights to the non-enabled access points of North Team.

Once you create it, the new role displays as an option in the Role list on each Add User or Edit User form.

To create a custom role:

1.   Go to Admin page > Custom Role tab.

2.   Click Add Custom Role (see Figure 22 on page 25).

3.   Name the custom role.

4.   Select portal access points that will be granted to users that are assigned this role. For more infor­mation on access points, see "Access Point Options" on page 25

5.   Click Save.

6.   Verify the new cus­tom role is an avail­able as an option on the Add User or Edit User page.

Table 1: Access Point Options

Access Point

Explanation

Users

Users access point enabled in an assigned role lets the user add/edit/delete users in their assigned group. By default, Admin Users can create, edit, or delete users at all organization levels.

Vehicles

Vehicles access point enabled in an assigned role lets the user add/edit/delete vehicles in their group. By default, Admin Users can create, edit, or delete vehicles at all organization levels.

Devices

Devices access point enabled in an assigned role lets the user edit/delete devices within their assigned group. By default, Admin Users can create, edit, or delete devices at all organization levels.

Zones

Zones access point enabled in an assigned role lets the user add/edit/delete zones within their assigned group. By default, Admin Users can create, edit, or delete Zones at all organization levels.

Red Flags

Red Flags access point enabled in an assigned role lets the user...

•   View/edit/delete red flag alerts that they create.

•   Assign alerts to groups, drivers and vehicles that are in their group hierarchy. If the alert is assigned to a vehicle type (light, medium, heavy), the vehicle will be further filtered by vehi­cles that are within the owners group hierarchy.

•   Send notification to any person in the account regardless of group hierarchy. This will allow the user to send notifications to supervisors/managers who are above them in the group hier­archy.

By default, Admin Users can...

•   View/edit/delete red flag alerts that they or any other users in their group hierarchy create.

•   Assign alerts to groups that are in the group hierarchy of the owner of the alert. If the alert is assigned to a vehicle type (light, medium, heavy), the vehicle will be further filtered by vehi­cles that are within the owners group hierarchy.

•   Send notification to any person in the account regardless of group hierarchy. This will allow the user to send notifications to supervisors/managers who are above them in the group hier­archy.

•   Change the owner of the alert to a user that is within their group hierarchy.

Reports

Reports access point enabled in an assigned role lets the user...

•   View/edit/delete report schedules that they create.

•   Report on groups, drivers and vehicles that are in their group hierarchy.

By default, Admin Users can...

•   View/edit/delete report schedules that they or any other users in their group create.

•   Report on groups, drivers and vehicles in the group hierarchy of the report schedule owner.

•   Change the owner of the report schedule to a user that is within their group hierarchy.

Organiza­tion

Organization access point enabled in an assigned role lets the user add teams within their assigned group. By default, Admin Users can add divisions and teams at all levels in organization.

Speed By Street

Speed By Street access point enabled in an assigned role lets the user request speed limit adjust­ments from inthinc.

Live Fleet

Live Fleet access point enabled in an assigned role lets the user configure Live Fleet map views within their assigned group or organization level.

Note: The map view will automatically zoom out as the fleet boundaries expand.

Crash Report

Crash Report access point enabled in an assigned role lets the user view crash reports from their assigned group. By default, Admin Users can edit or delete crash reports at all organization levels.

Forgive Events

Forgive Events access point enabled in an assigned role lets the user forgive an item in their assigned group.
