Introduction

Note: The tiwi™ device is not designed for crash reconstruction, just crash detection.

Crash Detection – A crash on the tiwiPro™ Portal device is defined as an impact event whose acceleration data exceeds the hardware and software thresholds.

Note: During a crash the vehicle bus is mostly concerned with firing the airbag, so it is common that OBD information during a crash drops out. The data prior to the accident should be accurate and will give a fairly good idea of what the driver was doing, but the point of impact may not exactly match the flag it in the crash data.

Rollover Detection – A rollover on the tiwiPro device is defined as the presence of a constant acceleration of at least 0.7G for twenty seconds in any axis except for the downward axis. The downward axis is the axis of gravity for a vehicle when it is parked.

Note: It is possible to have a crash and a rollover occur, so long as the criteria for each event are met.

What is Crash Data?

Crash Data in tiwiPro™ Portal is a 30 second snapshot of vehicle history sent when a crash or rollover event is detected. (The snapshot is 20 seconds pre-event and 10 seconds post). Crash data consists of the following information:

•   GPS Speed — Speed as measured by inthinc® device’s GPS.

•   OBD Speed — Speed as read by inthinc® device from the Vehicle bus.

•   GPS location

•   Time

•   RPM - Read from the vehicle bus.

•   Seatbelt Availability - If seat belt was being monitored by the inthinc® device. This depends on the vehicle type.

•   Seatbelt state - If available, whether the seat belt was buckled or unbuckled.

Note: If the device is only using GPS, no vehicle data will be gathered. OBD speed and RPM fields will be 0. If seat belt data is not available, the field will be NA.

What can I do with Crash History List?

Once the Crash History page displays, you can:

•   Customize the columns displayed in the table and have the Report remember them.

•   View the list of crashes in the Crash History table for your respective organizational level. Choose to view the history in the default layout or in a previously saved custom layout.

•   Search for reports by date, time, group, driver, vehicle, status, or weather.

•   Refresh the Report to get the latest data.

•   Share the table information with others (by way of e-mail, generated PDF, generated Excel spreadsheet).

•   View the details of a crash.

If you are a fleet or division manager, or if you have admin rights on the account, you can:

•   Add a Crash to the Report.

•   Edit a Crash.

•   Exclude a Crash.
